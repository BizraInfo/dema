# Boundary: Core Runtime vs DEMA Face

## The Absolute Law: No Shadow State
BIZRA operates under a strict separation of powers between the **Constitutional Runtime (BIZRA Omega)** and the **Sovereign Operator Surface (DEMA)**. 

**DEMA is the single visible face of the ecosystem.** It unifies the web shell, the CLI, and future desktop operator surfaces. However, it must **never** invent its own truth.

### 1. BIZRA Core (The Kernel)
- Owns all constitutional truth.
- Executes FATE gates (I1-I7 admissibility) locally.
- Generates cryptographically secure Receipts and Manifests (WebCrypto SHA-256 / BLAKE3 emulated).
- Operates the underlying PAT-7 (Personal Action) and SAT-5 (System Action) swarms securely behind the scenes.

### 2. DEMA (The Product Face)
- Reads the local Trust State safely via gateway contracts.
- Renders the Receipt Chain, Local Resource Registry, and Manifests.
- Packages the CLI (`dema status`, `dema ask`).
- Exposes the 6 Jurisdictional Modes (Ask, Code, Research, Browser, Computer, Trust).
- Displays the Trust Strip (Ihsān Floor, Next Admissible Action, Ledger Hash) truthfully.

**Rule:** DEMA does not duplicate mission law. It does not own final admissibility. It does not mutate chain truth outside approved gateway contracts. Dema renders exact, verified truth produced by the Bizra-Kernel.
