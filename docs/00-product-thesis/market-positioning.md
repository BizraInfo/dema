# DEMA Market Positioning

## Position statement

DEMA is the sovereign operator for people who need an AI assistant they can trust with real work — not just chat, but research with proof, code with context, and actions with receipts.

## Target user

Knowledge workers and technical leaders who:
- Need an AI that can act (not just advise)
- Require proof of what was done (not just output)
- Work across code, research, and operations daily
- Value sovereignty over convenience
- Want one tool, not a fragmented toolbox

## The 7 killer screens

1. **Onboarding / First Citizen** — identity, trust baseline, initial permissions
2. **Dashboard / Trust Strip** — current state at a glance, always honest
3. **Ask / Research** — citations, confidence, sources, next action
4. **Receipts / Manifest** — proof chain, what was done, what it means
5. **Resources / Local Node** — what DEMA can see and access
6. **Actions / Browser + Computer** — operator surfaces with permission control
7. **Settings / Permissions + Memory** — full control over DEMA's access and recall
