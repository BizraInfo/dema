# Security Policy

Dema is local-first and consent-bound.

## Non-negotiables

- No hidden background daemon.
- No action without explicit user consent.
- No secrets committed to the repo.
- No unsigned update channel.
- No workflow action refs using moving tags.
- No public network/federation claims before measured proof.

## Reporting

For private reports, contact the BIZRA maintainers through the official GitHub security advisory channel.

## Current security posture

Dema v0.1 is an alpha shell. It must be reviewed before production use.
